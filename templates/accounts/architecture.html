🎥 Camera Service Architecture

┌─ Camera Service (Port 8001) ─────────────────────────┐
│ │
│ API Endpoints: │
│ ├─ GET /api/cameras/ │
│ ├─ GET /api/cameras/<id>/feed/ │
│ ├─ GET /api/cameras/<id>/test/ │
│ ├─ GET /api/mobile-cameras/<id>/feed/ │
│ └─ GET /api/mobile-cameras/<id>/test/ │
│ │
│ CameraManager (Singleton): │
│ └─ Manages CameraStreamer instances │
│ └─ One background thread per camera │
│ │
│ CameraStreamer Thread Loop: │
│ 1. Connect to RTSP/HTTP stream │
│ 2. Read frame via OpenCV │
│ 3. Resize: Original → 640x360 │
│ 4. Encode: JPEG quality 60 │
│ 5. Store frame (thread-safe with lock) │
│ 6. Sleep 0.05s (~20 FPS) │
│ 7. Auto-reconnect on error │
│ 8. Auto-stop after 90s inactivity │
│ │
└───────────────────────────────────────────────────────┘

📦 Packet & Protocol Details

HTTP Requests
Protocol: HTTP/1.1
Size: 1-100 KB
Headers: Cookie, CSRF, Content-Type
WebSocket Messages
Protocol: WS/WSS
Format: JSON
Size: 1-5 KB per message
RTSP Stream
Protocol: RTSP → RTP/UDP
Codec: H.264
Bandwidth: 300-600 KB/s
MJPEG Stream
Protocol: HTTP/MJPEG
Format: multipart/x-mixed-replace
Frame Size: 15-30 KB
WebRTC Media
Protocol: RTP/SRTP over UDP
Codecs: VP8/VP9, Opus
Bandwidth: 500 KB/s - 2 MB/s
Database Queries
Protocol: SQLite/PostgreSQL
ORM: Django ORM
Response: <10ms (local)
