{
  "type": "chat",

5. DATABASE STRUCTURE

5.1 DATABASE CONFIGURATION
--------------------------------------------------------------------------------

Database Engine: SQLite3
Database File: db.sqlite3 (in project root)
Shared By: Main application and camera service

Connection Settings:
- ENGINE: django.db.backends.sqlite3
- NAME: BASE_DIR / 'db.sqlite3'
- Atomic transactions: Enabled by default
- Foreign key constraints: Enabled

Production Recommendations:
- PostgreSQL 12+ for better concurrency
- Connection pooling (pgbouncer)
- Separate read replicas for camera service
- Regular backups and WAL archiving

5.2 USER AND PROFILE MODELS
--------------------------------------------------------------------------------

Table: auth_user (Django built-in)
Fields:
  - id: INTEGER PRIMARY KEY AUTOINCREMENT
  - username: VARCHAR(150) UNIQUE NOT NULL
  - first_name: VARCHAR(150)
  - last_name: VARCHAR(150)
  - email: VARCHAR(254)
  - password: VARCHAR(128) NOT NULL (hashed)
  - is_staff: BOOLEAN DEFAULT 0
  - is_active: BOOLEAN DEFAULT 1
  - is_superuser: BOOLEAN DEFAULT 0
  - date_joined: DATETIME NOT NULL
  - last_login: DATETIME

Table: accounts_userprofile
Fields:
  - id: INTEGER PRIMARY KEY AUTOINCREMENT
  - user_id: INTEGER UNIQUE NOT NULL
  - user_type: VARCHAR(10) NOT NULL
    Choices: 'student', 'teacher'
  - bio: TEXT
  - phone: VARCHAR(20)
  - date_of_birth: DATE
  - address: TEXT
  - profile_picture: VARCHAR(100) (file path)
  - avatar_url: VARCHAR(500)
  - display_name: VARCHAR(100)
  - student_id: VARCHAR(20)
  - grade: VARCHAR(20)
  - enrollment_date: DATE
  - employee_id: VARCHAR(20)
  - department: VARCHAR(100)
  - specialization: VARCHAR(200)
  - join_date: DATE
  - linkedin: VARCHAR(200)
  - twitter: VARCHAR(200)
  - website: VARCHAR(200)
  - created_at: DATETIME
  - updated_at: DATETIME

Relationships:
  - user_id: FOREIGN KEY -> auth_user.id (CASCADE)

Indexes:
  - user_id (UNIQUE)
  - user_type

5.3 CAMERA MODELS
--------------------------------------------------------------------------------

Table: cameras_camera
Fields:
  - id: INTEGER PRIMARY KEY AUTOINCREMENT
  - name: VARCHAR(100) NOT NULL
  - rtsp_url: VARCHAR(500) NOT NULL
  - username: VARCHAR(100)
  - password: VARCHAR(100)
  - ip_address: VARCHAR(50) NOT NULL
  - port: INTEGER DEFAULT 554
  - stream_path: VARCHAR(200) DEFAULT '/stream'
  - is_active: BOOLEAN DEFAULT 1
  - created_at: DATETIME NOT NULL

Indexes:
  - is_active
  - ip_address

Table: cameras_camerapermission
Fields:
  - id: INTEGER PRIMARY KEY AUTOINCREMENT
  - camera_id: INTEGER NOT NULL
  - teacher_id: INTEGER NOT NULL
  - granted_by_id: INTEGER
  - granted_at: DATETIME NOT NULL

Relationships:
  - camera_id: FOREIGN KEY -> cameras_camera.id (CASCADE)
  - teacher_id: FOREIGN KEY -> auth_user.id (CASCADE)
  - granted_by_id: FOREIGN KEY -> auth_user.id (SET NULL)

Constraints:
  - UNIQUE(camera_id, teacher_id)

Indexes:
  - camera_id
  - teacher_id

5.5 MEETING MODELS
--------------------------------------------------------------------------------

Table: meetings_meeting
Fields:
  - id: INTEGER PRIMARY KEY AUTOINCREMENT
  - title: VARCHAR(200) NOT NULL
  - description: TEXT
  - teacher_id: INTEGER NOT NULL
  - meeting_code: VARCHAR(20) UNIQUE NOT NULL
  - scheduled_time: DATETIME NOT NULL
  - duration_minutes: INTEGER DEFAULT 60

  - allow_chat: BOOLEAN DEFAULT 1

  - teacher_id: FOREIGN KEY -> auth_user.id (CASCADE)

Indexes:
  - meeting_code (UNIQUE)
  - teacher_id
  - status

Table: meetings_meetingparticipant
Fields:
  - id: INTEGER PRIMARY KEY AUTOINCREMENT
  - meeting_id: INTEGER NOT NULL
  - user_id: INTEGER NOT NULL
  - joined_at: DATETIME
  - left_at: DATETIME
  - is_active: BOOLEAN DEFAULT 0

Relationships:
  - meeting_id: FOREIGN KEY -> meetings_meeting.id (CASCADE)
  - user_id: FOREIGN KEY -> auth_user.id (CASCADE)

Constraints:
  - UNIQUE(meeting_id, user_id)

Indexes:
  - meeting_id
  - user_id
  - is_active

5.7 DATABASE SCHEMA EXPLANATION
--------------------------------------------------------------------------------

User Management:
- auth_user: Core Django user model for authentication
- accounts_userprofile: Extended profile with role-specific fields
- OneToOne relationship ensures one profile per user
- user_type field determines role (teacher/student)

Camera Management:
- cameras_camera: RTSP camera configuration
- mobile_cameras_mobilecamera: Mobile camera configuration
- Separate tables for different camera types
- Permission tables implement many-to-many with metadata

Meeting System:
- meetings_meeting: Meeting configuration and metadata
- meetings_meetingparticipant: Tracks user participation
- meeting_code: Unique identifier for joining
- status field: Tracks meeting lifecycle

Permission System:
- CameraPermission: Grants teachers access to specific RTSP cameras
- MobileCameraPermission: Grants teachers access to mobile cameras
- granted_by field: Audit trail of who granted permission
- Unique constraints prevent duplicate permissions

Data Integrity:
- Foreign key constraints with CASCADE delete
- Unique constraints on critical fields
- Indexes on frequently queried fields
- Timestamps for audit trails
